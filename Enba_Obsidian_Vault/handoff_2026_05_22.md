## Supabase Şema Özeti

Migration sırası: v2 → v3 → v4 → ... → v25b + fix_rls_infinite_recursion

**Kritik tablolar:**
- `profiles` — kullanıcı profilleri, company_id, role
- `companies` — çok kiracılı yapı
- `financial_transactions` — MKodu taksonomi sistemi
- `tasks` — görev yönetimi
- `cashflow_entries` — nakit akışı
- `pnl_reports` — P&L raporları
- `production_logs` — üretim kayıtları
- `assets` — varlık takibi
- `hr_employees` — personel
- `parasut_tokens` — OAuth token saklama

**RLS:** Tüm tablolarda company_id bazlı satır güvenliği aktif. v24b ile temizlendi, sonsuz özyineleme v26'da çözüldü.

1. **scratch/schema_v1_full.sql vs supabase/schema.sql** — İki şema arasındaki fark incelenmeli. v1_full daha büyük (4.3 KB), güncel olan daha küçük (2.4 KB). Eksik tablolar var mı kontrol edilmeli.

2. **migration_v26_custom_holidays.sql** — Proje kökünde duruyor, `supabase/migrations/` altına taşınmadı. Uygulanmış mı kontrol et.

3. **migration_mcode_update.sql** — Proje kökünde, aynı şekilde migrations altına taşı.

4. **design_handoff klasörleri** — `design_handoff_detailed_plan/` ve `design_handoff_gorev_yonetimi/` aktif kullanımda mı yoksa arşivlenecek mi?
